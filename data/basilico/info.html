File structure

File contents:

  • molecular objects that represent the binding site and the ligands;
  • tables with analysis of the binding site ensemble;
  • this description.

Molecular objects

Molecular objects were modified to facilitate calculations in a number of ways:

  • only main alternative coordinate set was preserved;
  • multiple covalently bonded ligand molecules were merged together;
  • ICM objects were stripped;
  • non-relevant residues were removed;
  • protein residues were renumbered and merged into one chain (so that residue numbers are unique within the object and consistent across the objects);
  • atom tag gr was set for protein atoms (with values B, C or P);
  • residue tag stre was set for protein residues (with values in the format ligcl:type:stre(;ligcl:type:stre)*);
  • residue tag ligcl was set for ligand residues (with integer index of the ligand cluster).

Examples of object manipulation using these improvements are provided in the visualization scripts in the table headers.

Each amino acid residue has atoms of several groups:

B
backbone;
C
carbon (or non-polar) part of the side chain;
P
polar part of the side chain;
S
side chain (C + P);
T
total (or full residue) (B + S).

Side chain carbon/polar groups are defined as follows:

  • main rules:
    1. carbon atoms are non-polar;
    2. non-carbon atoms are polar;
  • additional rules:
    1. sulfur atoms of Met and Cys (in disulfides) are non-polar;
    2. carbon atoms are added to polar groups to avoid disjoint polar groups (Arg/Cζ, Asn,Asp/Cγ, Gln,Glu/Cδ, His/Cε1);
    3. carbon atom Cε is added to terminal polar group of Lys;
    4. carbon atom Cβ is added to single polar group of small side chain of Cys and Ser (for consistency Thr/Cβ is also polar).

Table tdef

Table tdef contains definition of the individual contact strength (see GPCR Dock 2010 paper).

Columns:

d
distance in Å;
stre
contact strength (in the range from 0 to 1).

Header contains various parameters to calculate contact strength, e.g. r_dHi is the upper limit on distance to consider given atom pair being in contact. Also contains macro dsStre to create molecular visualization of contact strength for the given protein residue group and ligand cluster; this macro is used in other tables.

Dependence of the contact strength on distance is illustrated on the built-in plot.

Table tlig

Table tlig contains information about ligands and ligand clusters.

Columns:

mol
chemical structure;
L
toggles ligand visualization on/off and indicates if the ligand is displayed (works in conjunction with the built-in actions);
obNm
object name;
molNm
molecule name;
rsNm
residue names;
lig
ligand index (corresponds to ligand tag);
ccl
chemical cluster (0 is specical value for singletons);
dcl
overlap cluster (0 is specical value for singletons);
cnt
number of ligands in the ligand cluster;
ligcl
ligand cluster (from 1).

To toggle ligand visualization on/off, change current row by clicking on any cell of the desired ligand cluster or use buttons in column L.

Table tob

Table tob contains correspondence between objects and ligands, in the original object order.

Columns:

obNm
object name;
lig
ligand index;
ligcl
ligand cluster.

Table tres

Table tres contains information about binding site residues.

Columns:

original
residue number before renumbering, possibly with insertion code;
resNum
residue number, possibly with insertion code;
ch
chain name, e.g. A1;
swiss
name of the associated sequence;
res
residue code.

Residue codes were taken from the associated sequence (if any) or that are most frequent in the objects.

Table tmut

Table tmut contains information about mutations relative to the provided sequence or the most frequent residue code in the given position and may be empty. Table tmut is not limited to binding site residues and thus preserves information required for extended SVG visualization mode with external residue selection.

Columns:

ligcl
ligand cluster;
ch
chain name, e.g. A1;
original
residue number before renumbering, possibly with insertion code;
res
residue code ('-' means deletion);
swiss
residue code in the reference sequence.

Header contains reference sequence(s), e.g. seqA1. Only useful for provided sequences since numeration is not preserved and assumed to start from 1.

Table tflex

Table tflex contains analysis of flexibility of the binding site residues.

Columns:

resNum
residue number, possibly with insertion code;
nonflex
residue non-flexibility, from 0 (all conformations are different) to 1 (the same conformation);
nonflexbb
residue non-flexibility, backbone only;
dbb
difference between backbone-only and full-residue non-flexibility.

Header contains the visualization action. Click on the row to display residue flexibility.

Table tstre

Table tstre contains contact strengths for all ligand clusters and binding site residues (groups).

Columns:

ligcl
ligand cluster;
res
residue code;
sel
residue number / contact type;
stre
contact strength, from 0 to tdef.r_streMax;
type
contact type;
rnumt
residue type + contact type (used for plot only).

Contact strengths are shown on the built-in plot. For visualization purposes residues are renumbered in the plot so that all residues are equidistantly placed on the x-axis, with contacts of B, C, P, S, T types (in that order).

Header contains the visualization action. Click on the row or contact on the plot to display contacts of the given selection / ligand cluster pair.

Table tcdf

Table tcdf contains CDFs of contact strength of three contact types (B, C, P).

Columns:

stre
contact strength;
freqX
frequency of contacts with the given strength of type X;
cdfX
CDF of contact strength of type X.

CDFs are shown on the built-in plot.

Table tfp

Table tfp contains contact strengths of the full set of residues/groups vs all ligand clusters.

Columns:

sel
residue number / contact type;
type
contact type;
streX
contact strength for ligand cluster X, from 0 to tdef.r_streMax.

Header contains the visualization action. Click on the row (streX columns) or double-click on the cell to display contacts of the given selection / ligand cluster pair.

Table CL

Table CL contains clustering of the ligand clusters by contact fingerprints. There are four different split types:

BCP
backbone/carbon/polar;
BS
backbone/side chain;
S
side chain;
T
full residue.

Contact fingerprints depend on split type, thus the table contains four different fingerprint clusters. Table header contains four distance matrices for ligand clusters in terms of dissimilarity of the contact fingerprints, and corresponding trees. See description of make tree command in the ICM manual.

Columns:

ligcl
ligand cluster;
fpclX
contact fingerprint cluster for split type X.

Table CLa

Table CLa contains clustering of the ligand clusters by contact fingerprints, see description of table CL. In contrast to table CL (which lists individual ligand clusters), table CLa lists individual fingerprint clusters and aggregated ligand clusters.

Columns:

fpcl
contact fingerprint cluster;
split
split type;
ligcl
ligand clusters;
size
number of ligand clusters in fingerprint cluster;
nofres
number of residues that make strong contacts with the ligands from the fingerprint cluster;
cons
empty string or 'X' for consensus ligands;
resNum
list of residue numbers.

Consensus ligand for the given split type is ligand clusters that belong to first 80% of fingerprint clusters, sorted by size and number of residues with strong contacts. This filter excludes unusual contact patterns.

Header contains macro dsCL to create molecular visualization. Click on the row to display 3D visualization of the fingerprint cluster.

Table tcont

Table tcont contains information about binding site residues that have at least one strong contact with consensus ligands.

Columns:

split
split type;
res
residue code;
sel
residue number / contact type;
type
contact type;
stre
average contact strength of strong contacts;
freq
frequency of strong contacts relative to all ligands;
w
residue weight (= strength * frequency);
ligcl
ligand clusters with strong contacts.

Contact strengths are displayed on the built-in plot.

Header contains the visualization action. Click on the table row or contact on the plot to display 3D visualization of the contact.
